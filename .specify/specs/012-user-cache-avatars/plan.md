# Implementation Plan: User Profile Cache with Comment Avatars

**Branch**: `012-user-cache-avatars` | **Date**: 2026-06-09 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `.specify/specs/012-user-cache-avatars/spec.md`

## Summary

Extend the existing Vuex `state.cache` with a `userProfiles` map (uid → profile) and a store
action that checks the map before fetching from Firestore. Update `Comment.vue` to call this
cached lookup and pass the result into the already-existing `useAvatar` composable, which
resolves civ flags, upload URLs, and initials — exactly as `Header.vue` already does.

## Technical Context

**Language/Version**: JavaScript (ES2022), Node 22 (functions only — not relevant here)

**Primary Dependencies**: Vue 3, Vuex 4, Vuetify 3, Firebase JS SDK 10 — all already present

**Storage**: Cloud Firestore (`users/{uid}` documents) — read-only from the frontend

**Testing**: Manual golden-path per constitution (no formal test suite)

**Target Platform**: Web (Vue SPA)

**Project Type**: Web application (Vue 3 SPA)

**Performance Goals**: Zero duplicate Firestore reads per session per uid (SC-001, SC-002)

**Constraints**: Must not break the existing `useAvatar` composable contract; must handle
the `{ type, ref }` avatar shape already used throughout the app.

**Scale/Scope**: ~4M reads/month currently at 2.7× free tier. Comments pages are
read-heavy; multiple comments per author are the common case. This cache directly reduces
that read count.
