# Implementation Plan: Icon Gap Sync

**Branch**: `017-icon-gap-sync` | **Date**: 2026-06-16 | **Spec**: [spec.md](spec.md)

## Summary

New second admin card added to `src/views/Admin.vue` (below the existing Sync Data card). The feature identifies AOE4World items absent from the 12 local JSON icon files in two ways: **new items** (no local entry) and **civ extensions** (entry exists but a specific civ is missing from its `civ[]` array). Admins assign items to categories and download updated JSONs (P2), then optionally fetch CDN icons, convert to WebP via Canvas API, and download a zip (P3). Adds JSZip as the only new runtime dependency. All processing is client-side; zero Firebase reads/writes.

## Implementation Design

### 1. Source Data Strategy

The card checks `sourceData.value` (populated by the Sync Data pipeline). If already loaded, no re-fetch is needed. If null, a "Fetch data first" notice is shown with a button that runs the same fetch sequence as `startSync()` (units + buildings + technologies + abilities in parallel). The two cards share the same `sourceData` ref.

### 2. Civ Slug → Local ShortName Mapping

A new constant `CIV_SLUG_MAP` in Admin.vue maps AOE4World civ slugs (used in API responses) to the 3-letter local codes used in `civ[]` arrays:

```js

  abbasid: "ABB",      ayyubids: "AYY",    byzantines: "BYZ",
  chinese: "CHI",      delhi: "DEL",       english: "ENG",
  french: "FRE",       golden_horde: "GOH",hre: "HRE",
  lancaster: "HOL",    japanese: "JAP",    jin: "JIN",
  jeanne: "JDA",       templar: "KTE",     macedonian: "MAC",
  malians: "MAL",      mongols: "MON",     dragon: "DRA",

  tughluq: "TUG",      zhuxi: "ZXL",
```

### 3. Gap Detection Algorithm (`buildGapList`)

```
Input: sourceData (already fetched)
Output: GapItem[]

allSourceItems = flatten [units, buildings, technologies, abilities] arrays
localAll = flatten all 12 CATEGORY_CONFIG.data arrays

// Pre-compute id-to-civCount for folder resolution (step 6)
civCountById = Map<id, Set<civSlug>>
for each item in allSourceItems:
  civCountById.get(item.id).add(item.civilization)

seen = Set<"id:civSlug">
gaps = []

for each sourceItem of allSourceItems:
  civSlug = sourceItem.civilization   // e.g. "ayyubids"
  civCode = CIV_SLUG_MAP[civSlug]
  if !civCode: continue               // civ not tracked locally
  gapKey = `${sourceItem.id}:${civSlug}`
  if seen.has(gapKey): continue
  seen.add(gapKey)

  localEntry = localAll.find(e => e.id === sourceItem.id || e.title === sourceItem.name)
  civCount = civCountById.get(sourceItem.id).size

  if !localEntry:
    gaps.push({ id, name, type, age, civ: civSlug, civCode, civCount,
                description, costs, baseId, exploreUrl,
                kind: 'new', localEntry: null, localCategory: null })

  else if !localEntry.civ.includes(civCode):
    localCat = CATEGORY_CONFIG.find(c => c.data.includes(localEntry))
    gaps.push({ ..., kind: 'civ-extension', localEntry, localCategory: localCat })

return gaps
```

```
// Many-civ: match functional group
```

### 6. Existing Icon Check — Client-Side (FR-015)

No direct filesystem access from the browser. Client-side implementation:

- **Civ extension**: `localEntry.imgSrc` is already set → skip image fetch; preserve `imgSrc`
- **New item**: call `findLocalIconByName(gapItem)` (already in Admin.vue) — checks all 12 categories for an entry with the same `id` or `name` that has a non-empty `imgSrc`. If found, mark as "icon exists", reuse the existing path in the JSON entry, skip CDN fetch.

### 7. Image Processing Pipeline (`processGapImage`)

```
async processGapImage(gapItem, assignment):
  // FR-015 checks
  if gapItem.kind === 'civ-extension': return { skip: true }
  existingPath = findLocalIconByName(gapItem)
  if existingPath:
    assignment.imgSrc = existingPath
    return { skip: true, reason: 'icon-exists' }

  // CDN fetch
  // type plural: unit → units, building → buildings, technology → technologies, ability → abilities
  typePlural = gapItem.type + 's'  // (adjust irregulars: ability → abilities ✓)
  url = `https://data.aoe4world.com/images/${typePlural}/${gapItem.civ}/${gapItem.id}.png`
  response = await fetch(url)
  if !response.ok: return { skip: false, error: `HTTP ${response.status}` }
  pngBlob = await response.blob()

  // Canvas resize + WebP conversion
  bitmapUrl = URL.createObjectURL(pngBlob)
  img = await loadImage(bitmapUrl)          // returns HTMLImageElement
  URL.revokeObjectURL(bitmapUrl)

  canvas = document.createElement('canvas')
  canvas.width = 48; canvas.height = 48
  canvas.getContext('2d').drawImage(img, 0, 0, 48, 48)
  webpBlob = await canvasToBlob(canvas, 'image/webp', 0.8)

  folderName = assignment.imageFolder        // admin-confirmed
  zipPath = `${folderName}/${gapItem.id}.webp`
  assignment.imgSrc = `/assets/pictures/${zipPath}`
  return { skip: false, blob: webpBlob, zipPath }
```

All icons are resized to **48 × 48 px** (hardcoded constant). No runtime dimension discovery.

### 8. State Management

New refs added to Admin.vue `setup()` under the `// === ICON GAP SYNC ===` banner:

```js
const gapPhase       = ref('idle')   // 'idle' | 'scanning' | 'results'
const gapItems       = ref([])        // GapItem[] — all detected gaps
const gapCivFilter   = ref(null)      // civ slug currently filtered, or null = all
const gapAssignments = ref({})        // { [gapKey]: { categoryKey, imageFolder, imgSrc, confirmed } }
const gapImageStatus = ref({})        // { [gapKey]: 'pending'|'processing'|'done'|'exists'|'skipped'|'error' }
const gapJsonSaved   = ref({})        // { [categoryKey]: boolean }
const gapZipSaved    = ref(false)
```

Derived computeds:

```js
const filteredGapItems = computed(() =>
  gapCivFilter.value
    ? gapItems.value.filter(g => g.civ === gapCivFilter.value)
    : gapItems.value
)
const gapCivOptions = computed(() =>
  [...new Set(gapItems.value.map(g => g.civ))].sort()
)
const canDownloadJsons = computed(() =>
  gapItems.value.some(g => gapAssignments.value[`${g.id}:${g.civ}`]?.confirmed)
)
const canDownloadImages = computed(() =>
  gapItems.value.some(g => {
    const a = gapAssignments.value[`${g.id}:${g.civ}`]
    return a?.confirmed && g.kind === 'new' && a.imageFolder
  })
)
```

### 9. JSON Download (`downloadGapJsons`)

Groups confirmed assignments by category key, then for each affected category:

```
for each categoryKey with at least one confirmed assignment:
  entries = deep-clone of CATEGORY_CONFIG[categoryKey].data
  
  for each new-item assignment in this category:
    entries.push({
      title: gapItem.name, civ: [gapItem.civCode],
      age: gapItem.age, id: gapItem.id, type: gapItem.type,
      description: gapItem.description, costs: gapItem.costs,
      exploreUrl: gapItem.exploreUrl, imgSrc: assignment.imgSrc || "",
    })
  
  for each civ-extension assignment in this category:
    idx = entries.findIndex(e => e.id === gapItem.id || e.title === gapItem.name)
    if idx !== -1:
      entries[idx].civ = [...entries[idx].civ, gapItem.civCode].sort()
  
  downloadObjectAsJSONFile(entries, CATEGORY_CONFIG[categoryKey].filename)
  gapJsonSaved[categoryKey] = true
```

### 11. Zip Assembly (`downloadGapImagesZip`)

```
zip = new JSZip()

for each confirmed new-item assignment with imageFolder:
  result = await processGapImage(gapItem, assignment)
  gapImageStatus[gapKey] = result.error ? 'error' : result.skip ? 'exists' : 'done'
  if result.blob:
    zip.file(result.zipPath, result.blob)

if zip has files:
  blob = await zip.generateAsync({ type: 'blob' })
  trigger download: 'icon-gap-sync.zip'

gapZipSaved = true
```
