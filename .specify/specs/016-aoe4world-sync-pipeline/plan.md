# Implementation Plan: AOE4World Sync Pipeline Improvement

**Branch**: `016-aoe4world-sync-pipeline` | **Date**: 2026-06-16 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/016-aoe4world-sync-pipeline/spec.md`

## Summary

Refactor the "Sync Data with AOE4WORLD" button in `Admin.vue` from a single-shot fire-and-download into a two-phase pipeline: a **fetch-and-preview** phase that surfaces matched counts and unmatched entries per category, and an **apply-and-download** phase gated behind manual resolution of any mismatches. All changes are client-side within `Admin.vue`; no new files, no new dependencies, no backend changes. Vuetify 3 native components cover all new UI.

## Implementation Approach

### Script changes (`<script>` section of `Admin.vue`)

1. **Add reactive state** — `syncPhase`, `fetchStatus`, `sourceData`, `categoryResults` as `ref` locals.

2. **Add `CATEGORY_CONFIG`** — static array of 12 entries mapping each category to its filename, source key(s), exploreType, and imported JSON reference (replaces the 24-line repetitive call block).

3. **Refactor `syncData` → `runMatchPass(sourceArray, localEntries)`** — pure function returning `{ matched: [], unmatched: [] }` without mutating anything. Matching logic unchanged: `source.id === local.id || source.name === local.title`.

4. **Refactor `updateImageMetaData` → `startSync()`**:
   - Sets `syncPhase = 'fetching'`
   - Fires all 4 fetches in parallel (`Promise.allSettled`) with per-source status updates
   - On settle: runs `runMatchPass` per category, populates `categoryResults`, transitions to `'preview'`

5. **Add `resolveEntry(categoryKey, localEntry, sourceEntry)`** — moves an entry from `unmatched` to `resolved`.

6. **Add `skipEntry(categoryKey, localEntry)`** — moves an entry from `unmatched` to `skipped`.

7. **Add `applyAndDownload()`** — for each non-errored category, builds enriched JSON from `matched + resolved` (with original data for `skipped`), calls `downloadObjectAsJSONFile`, transitions to `'idle'`.

8. **Add `canDownload` computed** — `true` when `syncPhase === 'preview'` and no non-errored category has remaining `unmatched` entries.

### Template changes (`<template>` section of `Admin.vue`)

Replace the existing single-button card with a conditional panel stack inside the same `v-col`:

- **Idle state**: Original button "Sync Data with AOE4WORLD" (unchanged appearance).
- **Fetching state**: Fetch progress section — 4 rows, each showing source name + status chip (`v-chip` with loading/success/error variant).
- **Preview state**:
  - Per-category summary rows: category name, matched count, error badge if errored.
  - For each `unmatched` entry in a category: entry title + `v-autocomplete` dropdown (items = source array for that category, item-title = 'name', filter = case-insensitive substring ≥ 2 chars) + "Skip" button.
  - "Apply & Download" `v-btn` (disabled if `!canDownload`).
- **Downloading state**: Brief "Downloads triggered…" message, auto-resets to idle.
